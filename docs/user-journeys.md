# Troglodyte Works User Journeys

Version: 1.0

---

# Purpose

This document defines the major journeys people may take through Troglodyte Works.

A Journey is not a page.

A Journey is the path someone follows as they move from uncertainty toward confidence.

Pages, buttons, guides, services, and tools exist to support Journeys.

---

# Core Idea

Troglodyte Works should not force people to understand the platform before receiving value.

The experience should begin with where the person is starting.

The first interaction should feel like a conversation:

Every community starts somewhere.

Let's find yours.

---

# Identity Model

## Visitor

A Visitor has no account.

They may:

- Explore
- Follow an advertisement link
- Arrive from search
- Preview what Troglodyte Works can do

A Visitor becomes a Member after authenticating.

---

## Member

A Member has an identity.

A Member may:

- Join communities
- Own communities
- Manage servers
- Participate in events
- Vote in polls
- Use guides
- Connect services

Membership does not automatically mean payment.

Billing and subscription status are separate from identity.

# Journey 1: Join Friends

## Starting Point

Someone arrives because they were invited.

They may come from:

- Discord
- Steam
- An invite code
- A shared link
- A friend

## First Question

How did your friends invite you?

## Possible Paths

- Continue with Discord
- Enter an invite code
- Connect with Steam
- Sign in with Google or email

## Goal

Help the person join the correct community with as little friction as possible.

## Success State

The Member arrives in the community workspace and understands what they can do next.

## Design Notes

Do not make the person learn Troglodyte Works vocabulary first.

They are not joining "a platform."

They are joining their people.

# Journey 2: Manage Game Servers

## Starting Point

Someone wants help with game servers.

They may already have a server, or they may want to create one.

## First Question

Are you already running a game server?

## Possible Paths

- I already have a server.
- I want to start a new server.
- I'm not sure yet.

## If They Already Have A Server

Ask where it is hosted.

Possible answers:

- Self-hosted
- Nitrado
- Another provider
- I don't know

## If They Want A New Server

Guide them toward selecting:

- Game
- Map or world
- Community
- Hosting needs
- Difficulty or setup level

## Goal

Help the Member manage or create a game server without needing to become a systems administrator.

## Success State

The server is connected, created, or clearly scoped as the next step.

## Design Notes

Avoid technical setup language too early.

Do not start with ports, config files, RCON, or command lines.

Reveal those only when they become useful.

# Journey 3: Explore Troglodyte Works

## Starting Point

Someone is curious but not ready to join or create anything.

## First Question

Would you like to see how Troglodyte Works helps a community?

## Possible Paths

- View a sample community
- Explore server tools
- See how Discord integration works
- Learn what Guides can do

## Goal

Let the Visitor experience value without pressure.

## Success State

The Visitor understands what Troglodyte Works can help them accomplish.

## Design Notes

Do not force signup too early.

Exploration should feel like being invited in, not being sold to.

---

# Journey 4: Create A Community

## Starting Point

A Member wants to gather people around a game, server, creator world, or shared project.

## First Question

What kind of community are you creating?

## Possible Paths

- Friends playing together
- Discord gaming group
- Streamer or creator world
- Clan or guild
- Family or private server

## Goal

Help the Member create a community space that can later connect servers, Discord, events, and automations.

## Success State

The Community exists and the Member has a clear next step.

## Design Notes

Community creation should come before technical setup.

The person should feel like they are starting something meaningful, not filling out a database form.

---

# Journey 5: Community Owner

## Starting Point

A Member owns or manages a Community.

## Common Goals

- Invite members
- Assign roles
- Connect Discord
- Create announcements
- Schedule events
- Create polls
- Connect servers
- Review activity

## Goal

Help the Community Owner coordinate people, servers, and events from one calm workspace.

## Success State

The Community Owner feels organized and in control.

## Design Notes

Do not overload the workspace with every possible admin option.

Show the next useful action.

Advanced tools should remain available but secondary.

---

# Journey 6: Community Member

## Starting Point

A Member belongs to a Community but does not manage it.

## Common Goals

- View announcements
- Join events
- Vote in polls
- See server status
- Read rules
- Invite friends if allowed
- Get help from a Guide

## Goal

Help the Member participate without exposing admin complexity.

## Success State

The Member knows what is happening and how to participate.

## Design Notes

Community Members should not see management tools unless their role allows it.

Their experience should feel simple, social, and clear.

---

# Journey 7: Guide-Assisted Help

## Starting Point

A Member needs help.

They may not know what to ask.

## Guide Behavior

The Guide should:

- Offer help without taking over
- Explain only as much as needed
- Adapt to the Member's behavior
- Suggest another Guide style when appropriate
- Never make the Member feel incapable

## Goal

Help the Member complete a task while increasing confidence.

## Success State

The Member understands what happened and feels capable of continuing.

## Design Notes

Guides are optional.

The product must remain excellent without them.

---

# Journey Design Rules

Every Journey should:

- Begin with the Member's situation
- Ask one clear question at a time
- Preserve context
- Avoid platform jargon
- Reveal complexity gradually
- Provide value before asking for payment
- Leave the Member more confident

---

# Product Reminder

Troglodyte Works is not asking:

What features should we show?

Troglodyte Works is asking:

Where is this person starting, and what is the next useful step?
